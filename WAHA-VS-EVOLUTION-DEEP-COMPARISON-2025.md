```

  ‚îî‚îÄ‚îÄ /dealer-c/webhook  ‚Üí Forward to :3003

**Pros:**
- ‚úÖ **Complete isolation** (1 crash ‚â† all down)
- ‚úÖ **Simple** (just spin up containers)
- ‚úÖ **Flexible** (different engines per dealer if needed)

**Cons:**
- ‚ö†Ô∏è **More containers** (3 dealers = 3 containers)
- ‚ö†Ô∏è **Manual scaling** (you manage container lifecycle)
- ‚ö†Ô∏è **Port management** (need unique ports)

#### Evolution API Multi-Tenant:
**Approach:** Single instance, multiple "instances" in DB

```
Evolution API (Single Container)
  ‚Üì

  ‚îú‚îÄ‚îÄ Instance: dealer-a (instanceId in all tables)
  ‚îú‚îÄ‚îÄ Instance: dealer-b
  ‚îî‚îÄ‚îÄ Instance: dealer-c

  POST /instance/create  ‚Üí New dealer
  POST /instance/{name}/sendText  ‚Üí Send for dealer

**Pros:**
- ‚úÖ **True multi-tenant** (1 container, N dealers)
- ‚úÖ **Auto-scaling** (just add instances via API)
- ‚úÖ **Centralized** (1 database, 1 API)
- ‚úÖ **Better for >10 dealers**

**Cons:**
- ‚ö†Ô∏è **Single point of failure** (1 crash = all down)
- ‚ö†Ô∏è **Shared resources** (1 dealer spam = affect others)
- ‚ö†Ô∏è **Database dependency** (PostgreSQL required)

### 8. BAN MITIGATION STRATEGIES (For Both)

**CRITICAL STRATEGIES for 2025:**

#### 1. Number Warming (ESSENTIAL)
```
Day 1-3:  Send/receive with real contacts (friends/family)
          Max 10 messages/day

Day 4-7:  Gradually increase to 20 messages/day
          Mix of sent + received

Day 8-14: Ready for light business use
          Max 50 messages/day

Day 15+:  Can scale to 100 messages/day
```

#### 2. Response Rate Management (TARGET: >30%)
```
For every 100 messages sent:
  ‚Üí At LEAST 30 customers should reply
  ‚Üí Ideally 50+ replies (50% rate)

How to achieve:
  ‚úÖ Ask questions in first message
  ‚úÖ Use customer names (personalization)
  ‚úÖ Include clear CTA ("Reply with YES if interested")
  ‚úÖ Offer opt-out ("Reply STOP to unsubscribe")
```

#### 3. Message Personalization
```javascript
// BAD (same message to everyone)
"Hallo! Wij hebben een aanbieding!"

// GOOD (personalized)
"Hallo [Naam]! Jij vroeg vorige week naar een [Car Model].
We hebben nu een mooie [Car Model] beschikbaar. Interesse?"
```

#### 4. Volume Limits (Per Number)
```
New number (0-14 days):   Max 20 msg/day
Warmed (15-30 days):      Max 50 msg/day
Established (30+ days):   Max 100 msg/day

NEVER exceed 100 messages/day per number!
```

#### 5. Multiple Numbers Strategy
```
Instead of:
  1 number √ó 300 messages/day = HIGH BAN RISK ‚ùå

Do:
  3 numbers √ó 100 messages/day = LOWER RISK ‚úÖ
```

#### 6. Avoid Detection Patterns
```
‚ùå DON'T:
- Send same message to >20 people
- Use bulk messaging features
- Send without warming up
- Ignore response rates
- Use brand new numbers

‚úÖ DO:
- Vary message content
- Space out messages (1-2 min apart)
- Warm up numbers properly
- Monitor response rates
- Use established numbers (>30 days)
```
